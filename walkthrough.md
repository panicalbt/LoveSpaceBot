# Готовый проект LoveSpace

Я успешно создал и настроил гибридную архитектуру для Telegram Mini App, где фронтенд использует легкий Vanilla HTML/JS, а бэкенд построен на бессерверных функциях Vercel и базе данных Postgres.

## Что было сделано

### 1. Архитектура и База Данных
- Инициализирован Node.js проект с **Prisma**.
- Создана схема БД (`prisma/schema.prisma`), включающая модели для `User`, `Couple`, `Task`, `Wish` и `Plan`.
- Настроено подключение к Vercel Postgres через переменные окружения.

### 2. Backend API (Vercel Serverless)
Создана папка `api/`, в которой размещены следующие Serverless функции:
- `api/auth.js`: Обрабатывает `initData` от Telegram, верифицирует криптографическую подпись с помощью вашего `TELEGRAM_BOT_TOKEN`, регистрирует пользователя и возвращает его стейт.
- `api/couple.js`: Логика создания уникального 6-значного кода пары и присоединения к ней.
- `api/tasks.js`, `api/wishes.js`, `api/plans.js`: Обработчики получения (GET) и создания/редактирования (POST) контента внутри пары.

### 3. Frontend (Single-Page HTML)
В папке `public/` размещен `index.html`:
- Никаких тяжелых фреймворков. Используется нативный `fetch` для связи с `/api/`.
- Реализованы экраны авторизации, главного дашборда со сводкой, экраны планов, заданий, желаний и настроек.
- Переключение тем (Темная/Светлая) и 4-х цветовых акцентов (Розовый, Красный, Синий, Зеленый).
- Интерактивный фон из партиклов на `Canvas` + `DeviceOrientation API` (реагирует на гироскоп телефона).

### 4. Настройка Деплоя
- Создан файл `vercel.json`, который указывает Vercel маршрутизировать все запросы `/api/*` в папку `api/`, а все остальные отдавать как статику из папки `public/`.

## 🚀 Инструкция по деплою на Vercel

Чтобы запустить это приложение в реальном мире, выполните следующие шаги:

1. **Запушьте код в GitHub:** 
   Создайте репозиторий и отправьте туда текущую папку `LoveSpace`.

2. **Подключите Vercel к GitHub:**
   - Зайдите на [Vercel.com](https://vercel.com) и нажмите `Add New -> Project`.
   - Выберите ваш репозиторий `LoveSpace`.
   - Во вкладке **Storage** (или прямо при создании проекта) добавьте **Vercel Postgres** базу данных.

3. **Настройте переменные окружения (Environment Variables):**
   Убедитесь, что в панели Vercel добавлены следующие переменные (Vercel Postgres добавит часть из них автоматически):
   - `POSTGRES_PRISMA_URL`
   - `POSTGRES_URL_NON_POOLING`
   - `TELEGRAM_BOT_TOKEN` — вставьте сюда токен вашего бота от `@BotFather` (необходим для валидации Telegram-подписей).

4. **Выполните миграцию БД:**
   Поскольку это первый запуск, вам нужно создать таблицы. Локально вы можете выполнить:
   `npx prisma db push` (убедившись, что у вас есть `.env` с ссылками на БД Vercel).
   *Альтернативно: добавьте `"postinstall": "prisma generate && prisma db push"` в ваш `package.json`, чтобы Vercel делал это сам при сборке.*

5. **Готово!**
   Скопируйте URL вашего Vercel-проекта (например, `https://lovespace.vercel.app`) и вставьте его в настройки `Web App` вашего Telegram-бота в `@BotFather`.
