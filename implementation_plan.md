# Telegram Mini App для пар: Промпт и План разработки

Это приложение позволит любовным парам планировать свое время, выполнять задания друг друга за баллы и тратить их на желания. Приложение будет иметь современный дизайн со светящимися эффектами, реагирующими на гироскоп партиклами, и поддержкой смены тем.

## User Review Required

Ниже представлены **готовый промпт** (который вы можете использовать в работе с любой LLM или передать мне для дальнейшего написания кода) и **пошаговый план разработки**. 
Пожалуйста, ознакомьтесь с планом. Одобрите его, если вы согласны с архитектурой (Vanilla HTML для фронтенда и Vercel Serverless + Postgres для бэкенда).

## Open Questions

1. **Архитектура бэкенда:** Поскольку фронтенд будет на чистом HTML/JS, а деплой и БД на Vercel, предлагается использовать **Vercel Serverless Functions** (папка `api/` с Node.js скриптами). Согласны?
2. **База данных:** Использовать **Vercel Postgres** совместно с **Prisma ORM** (или Drizzle) внутри Serverless функций для удобной работы с таблицами?
3. **Безопасность:** Авторизация через `Telegram WebApp initData` будет проверяться на бэкенде Vercel при каждом API запросе. Устраивает ли вас такой подход?

## 📝 Промпт для разработки

Вы можете скопировать этот промпт и использовать его как контекст для ИИ:

```text
Действуй как опытный Fullstack-разработчик. Создай Telegram Mini App для любовных пар со следующим функционалом и требованиями:

СУТЬ ПРИЛОЖЕНИЯ:
- Составление совместных планов на день/неделю/месяц.
- Выдача заданий друг другу. За выполнение заданий начисляются баллы.
- Магазин желаний: накопленные баллы можно тратить на "желания".

МЕХАНИКА ПАРЫ:
- Создание пары происходит по реферальному коду из 6 символов (заглавные буквы и цифры). Один генерирует код, второй его вводит.

АВТОРИЗАЦИЯ И БЕЗОПАСНОСТЬ:
- Строго через Telegram Web App API (TMA API). Данные пользователя (initData) отправляются на бэкенд и там валидируются (crypto/hmac-sha256).

ДИЗАЙН И UI:
- Современный, "дорогой" интерфейс с плавными анимациями и эффектами свечения (Glow эффекты).
- Переключение между темной и светлой темой.
- Выбор цветового акцента пользователем: красный, розовый, синий, зеленый.
- Динамический фон: светящиеся партиклы (частицы), которые реагируют на движения устройства с помощью гироскопа.
- Иконки: использовать исключительно бесплатный набор "coolicons".

ТЕХНИЧЕСКИЙ СТЕК И ДЕПЛОЙ:
- Frontend: Чистый HTML5, CSS3 (переменные для тем) и Vanilla JavaScript. Никаких тяжелых фреймворков на фронте.
- Backend: Vercel Serverless Functions (Node.js в папке /api).
- База данных: Vercel Postgres. Инструмент для работы с БД — Prisma ORM.
- Хостинг и деплой: Vercel (он отлично хостит и статику HTML, и серверные функции).

ЗАДАЧА:
1. Напиши схему базы данных (Prisma schema), включающую модели: User, Couple (связь двух юзеров), Plan, Task, Wish.
2. Подготовь код фронтенда (`index.html`) с интерактивным фоном (Canvas + гироскоп) и CSS-переменными для акцентов и тем.
3. Напиши логику связывания пары по 6-значному коду, вызывая API-эндпоинт Vercel.
4. Напиши пример Vercel Serverless функции (`api/couple.js`), которая будет создавать пару в Postgres и валидировать данные Telegram.
```

## 🛠 План реализации (Implementation Plan)

### Этап 1: Инициализация проекта для Vercel
- Инициализация `package.json` для бэкенд-зависимостей (`prisma`, `@vercel/postgres`, крипто-утилиты).
- Создание папки `public/` (или корня) для `index.html` и статики.
- Создание папки `api/` для серверных функций Vercel.

### Этап 2: Архитектура базы данных (Prisma + Vercel Postgres)
- Настройка `schema.prisma`.
- **Модели:**
  - `User` (telegramId, имя, тема, акцентный цвет, баланс баллов, role в паре).
  - `Couple` (уникальный 6-значный код, связь с двумя `User`).
  - `Plan` (тип: день/неделя/месяц, текст, статус выполнения).
  - `Task` (название, стоимость в баллах, автор, исполнитель, статус).
  - `Wish` (название, цена в баллах, статус покупки/исполнения).
- Подключение к БД и генерация клиента.

### Этап 3: Разработка Frontend (HTML/JS/CSS)
- Базовая разметка `index.html`.
- Подключение Telegram Web App скрипта.
- Реализация CSS-переменных (`:root`) для акцентных цветов и темной/светлой темы.
- Разработка компонента `BackgroundParticles` (Canvas + `DeviceOrientation API`).

### Этап 4: Разработка Backend (Vercel Serverless)
- Создание утилиты для криптографической проверки `initData` от Telegram.
- `api/auth.js` — получение данных пользователя, регистрация или возврат существующих данных.
- `api/couple.js` — генерация/присоединение по 6-значному коду.
- `api/tasks.js`, `api/wishes.js`, `api/plans.js` — CRUD операции для функционала.

### Этап 5: Интеграция Frontend с Backend
- Написание JS-функций `fetch` в `index.html` для обращения к `/api/...`.
- Логика авторизации: при старте отправляем `initData` на `/api/auth`, получаем стейт.
- Обновление интерфейса при покупке желаний и выполнении заданий.

### Этап 6: Деплой на Vercel
- Настройка `vercel.json` (маршрутизация, если необходимо).
- Деплой проекта через Vercel CLI или GitHub.
- Настройка переменных окружения (Postgres URL, Telegram Bot Token) в панели Vercel.
- Финальное тестирование в Telegram-боте.
